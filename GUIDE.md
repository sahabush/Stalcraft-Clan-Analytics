## Начало работы

### Приглашение бота

1. Перейдите по ссылке: [Пригласить бота SCWA](https://discord.com/api/oauth2/authorize?client_id=1487403880991031450&permissions=274878024704&scope=bot%20applications.commands)
2. Выберите сервер и подтвердите разрешения.
3. Бот готов к работе.

| Команда | Назначение |
|:---|:---|

## Авторизация

Для работы боту необходим доступ к вашему аккаунту EXBO (Stalcraft).

1. Введите `/auth`.
2. Перейдите по ссылке «Авторизоваться через Discord».
3. Подтвердите доступ в Discord.
4. Вы будете перенаправлены на сайт EXBO — подтвердите доступ там.
5. После успешной авторизации появится страница с подтверждением.

Авторизация сохраняется надолго и не требует повторного входа.

> Процесс может занимать больше времени, чем обычно, из-за задержек на сторонних сетевых узлах, не зависящих от Сервиса.

## Выбор клана

1. Введите `/clan select`.
2. Введите точное название клана.
3. Бот проверит через API ваше членство в клане.
4. При успешной проверке бот покажет ваш ранг и подтвердит привязку.

Руководство клана (LEADER, COLONEL) получает расширенные права управления скриншотами.

## Загрузка скриншотов

### Требования к скриншоту

Скриншот должен содержать **только таблицу результатов CW** — без элементов интерфейса, чата и посторонних деталей. Рекомендуется делать полный скриншот (F2) и обрезать его без потери качества.

### Загрузка

1. Введите `/add`.
2. Прикрепите файл скриншота (PNG, JPG, WebP).
3. Отправьте команду.

Бот обработает изображение и покажет результат. Обработка обычно занимает несколько секунд.

---

## Проверка и подтверждение

После обработки отображается таблица с распознанными данными. Для каждого игрока указана уверенность распознавания:

| Уверенность | Значение |
|:---|:---|
| > 90% | Данные корректны |
| 75-90% | Возможны незначительные ошибки |
| < 75% | Требуется проверка |

### Доступные действия

- **Подтвердить** — сохранить данные в статистику клана. При подтверждении можно указать карту, тир противника и результат боя. Все поля опциональны.
- **Редактировать** — исправить данные отдельного игрока перед сохранением.
- **Отклонить** — удалить скриншот без сохранения.

### Рейтинг при подтверждении

При указании тира противника (S/A/B/C/D/E) игрокам начисляется рейтинг по формуле:
> **(убийства + помощь) × множитель тира**

Множители: S = 5.0, A = 3.0, B = 2.0, C = 1.0, D = 0.7, E = 0.5.

## Просмотр статистики

### Полная таблица (`/table_full`)

Показывает статистику всех игроков клана за выбранный период. Поддерживает фильтрацию по карте и тиру противника.

**Параметры:**
- `days` — количество дней (по умолчанию 30)
- `map_name` — фильтр по карте (Малая Бердовка, Хвойник, Низина)
- `enemy_tier` — фильтр по тиру противника (S/A/B/C/D/E)

**Столбцы таблицы:**

| Столбец | Описание |
|:---|:---|
| # | Место по среднему счёту |
| Игрок | Никнейм |
| Игр | Количество скриншотов с участием игрока |
| Счёт | Средний счёт за игру |
| KD | Соотношение убийств к смертям |
| У/С/П | Суммарные убийства/смерти/помощь |
| Посещ | Сыграно игр / всего скриншотов |
| Статус | 🟢 играл / 🟡 пропускал / 🔴 не играл |

---

## Статистика игрока (`/player`)

Показывает детальную статистику игрока с историей.

**Параметры:**
- `days` — количество дней
- `nickname` — ник игрока (с автодополнением)

**Отображаемая информация:**
- Активность: количество игр, посещаемость, среднее место
- Боевые показатели: K/D, KDA, суммарные убийства/смерти/помощь
- Рекорды: лучший счёт, лучшее место, лучший ранг
- Прогресс-бары: счёт относительно максимального, K/D, посещаемость
- История счёта и K/D (ASCII-графики)
- Тренд за последние 3 игры

---

## Рейтинговая система (`/table_rating`)

Показывает рейтинговую таблицу игроков. Учитываются только матчи, в которых был указан тир противника.

**Столбцы таблицы:**

| Столбец | Описание |
|:---|:---|
| № | Место по рейтингу |
| Игрок | Никнейм |
| Рейтинг | Суммарный рейтинг |
| Матчей | Количество рейтинговых матчей |
| K+A | Убийства + помощь в рейтинговых матчах |
| KD | Общее соотношение убийств к смертям |
| Счёт | Средний счёт |

**Множители тиров:** S = 5.0x, A = 3.0x, B = 2.0x, C = 1.0x, D = 0.7x, E = 0.5x.

## История скриншотов

Доступна через `/screenshots` → «История». Позволяет просматривать скриншоты по одному с полной информацией:

- Дата и статус скриншота
- Карта, тир противника, результат боя
- Полная таблица игроков
- Посещаемость со списком отсутствовавших

**Навигация:** кнопки «Предыдущий» и «Следующий» для перелистывания.

**Действия:** экспорт текущего скриншота в CSV, удаление из статистики.

## Управление скриншотами

Доступно только лидерам (LEADER) и полковникам (COLONEL) через `/screenshots`.

| Действие | Описание |
|:---|:---|
| Список скриншотов | Все загруженные скриншоты с датами и статусами |
| История | Поскриншотный просмотр с навигацией |
| Удалить скриншот | Выбор и удаление конкретного скриншота |
| Удалить дубликаты | Автоматическое удаление повторяющихся скриншотов |
| Очистить все данные | Полное удаление всей статистики клана |
| Отклонить неподтверждённые | Массовое отклонение ожидающих скриншотов |
| Экспорт в Excel | Сохранение выбранного скриншота в CSV |

---

## Экспорт данных

Доступен через `/screenshots` → «Экспорт в Excel» или через кнопку «CSV» в истории скриншотов.

Файл содержит все данные скриншота: позиции, ники, убийства, смерти, помощь, казну, счёт, ранг, K/D, KDA. Открывается в Excel, Google Sheets и LibreOffice.

## Уведомления

## Частые вопросы

### Бот не распознал мой ник

Бот использует актуальный список участников клана из API Stalcraft. Если вы недавно вступили в клан, попросите руководство обновить список через `/clan select`.

### Долгая обработка скриншота

Обычно обработка занимает 3-10 секунд. Если прошло больше минуты — вероятно, бот перегружен. Попробуйте позже.

### Как часто обновляется список кланов

Список всех кланов обновляется раз в 24 часа. Список участников вашего клана — при каждом выборе клана.

### Бот не отвечает

Проверьте статус бота. Если статус «Не беспокоить» — идут технические работы. Попробуйте переавторизоваться через `/auth`.

### Безопасность данных

Подробно описано в [Политике конфиденциальности](https://docs.scwa.ru/privacy). Пароли не хранятся, скриншоты удаляются после обработки, токены шифруются.
